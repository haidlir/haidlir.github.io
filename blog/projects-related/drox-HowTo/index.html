Drox SDN Application - HowTo

Fri, Jul 24, 2015

Drox is a software defined networking (SDN) application built on pox controller. It is expected to control unicast traffic on flat L2 ethernet network with arbitrary topology. It uses multipath routing for the forwarding decision of each flow, hence it doesn’t transform the topology into a spanning tree or loop/circuit-free topology. A flow is combination of source-destination IP address and source-destination TCP/UDP port in the packet’s header. Packets associated in a flow are forwarded in a same path, where path is a sequence of switch/datapath between source and destination node. This application is created refering to “Enabling Mutlipath Routing for Unicast Traffic in Ethernet Network” paper. The rest of the artice will explain how to use drox.

Since drox is built on pox controller, you should install/copy/clone pox controller into your machine. You can find pox’s repository with all its documentations in here. Once your pox is working, you can download/clone/copy drox repository and place it into the ext directory inside the pox directory.

cd [pox-directory]/ext
git clone http://www.github.com/haidlir/drox/drox.git

To run drox sdn application, just execute run.sh. That shell script (run.sh) makes pox.py to call main.py (in drox directory) and the dependencies.

cd drox
chmod +x run.sh
./run.sh

Since drox also provides Dynamic Host Control Protocol (DHCP) server, all end-hosts in the topology which handled by drox, should be configured to use dynamic ip address. For convenient, you can use Mininet, a network simulator complemented with Open vSwitch, so you can connect all switches to controller (drox). You can download the mininet topology script here for example. You can also find information about mininet here.

Before run the topology, make sure that you have change the controller ip address, that mininet refers to, into the appropriate ip address.

...
    net = Mininet(Abilene(), switch=OVSKernelSwitch, controller=None)
    net.addController(RemoteController( name='c0', ip='[controller's ip address]' ))
...

Then, run the topology.

sudo python abilene.py
